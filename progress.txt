## Codebase Patterns
- Lambda Python code lives in `lambda/` subdirectory, independent of the TypeScript project
- Use `uvx ruff check` and `uvx ruff format` for Python linting (ruff is not installed globally, use uvx to run it)
- SAM template uses `Globals` section for shared function config (runtime, memory, timeout)
- Lambda handler returns `{ "statusCode": N, "body": json.dumps({...}) }` pattern
- `src/lib/env.ts` exports a validated config object — import it as `import config from "@/lib/env"` to access environment variables
- `dotenv` is already a dependency; env.ts calls `dotenv.config()` at import time
- All SDK clients (MilvusClient, LambdaClient) use lazy initialization — never instantiate at module level
- Next.js build evaluates API routes at build time; skip env validation during build via `NEXT_PHASE` check
- `src/lib/lambda.ts` exports `generateEmbedding(text)` — invoke AWS Lambda to get OpenAI embeddings; import as `import { generateEmbedding } from "@/lib/lambda"`
- Lambda response follows `{ statusCode, body }` pattern where body is a JSON string containing `{ embedding, model, dimensions }`
- Semantic Scholar `Paper` type includes `title` (always returned by default by S2 API) — added in US-008
- Next.js 14+ App Router dynamic route params are `Promise<{...}>` — must `await params` before accessing values
- For debounced search in React: use `useEffect` with `setTimeout` (300ms) + `AbortController` cleanup to cancel in-flight requests
- `src/app/page.tsx` is a `"use client"` component — all UI interactivity (search, vectorization, semantic search) lives here
- Semantic search API: `GET /api/papers/semantic-search?q={query}` returns `SemanticSearchResult[]` with `score` as 0-1 float
